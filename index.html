HOW WEB PAGES WORK

I will begin by defining some terms I will use frequently in this essay.
http is the primary protocol of the web, which is used by browsers and serves
to transfer content through the web.
The HTML is a simple programming language based on tags that provide instructions
for creating web pages such as where to place an image or which text to be made large or
which one to be put in italics. Web browsers could understand and interpret HTML and display
the web page to the viewer.

The steps of a Webpage being displayed

  1. A request is made when the link is clicked
  2. The page and it's resources are downloaded (Respond)
  3. The web browser uses the page resources to build the page
  4. The page is then rendered (displayed) to the user

REQUEST

The computer that handles these tasks are called DNS servers, what they do basically is translating
domain names which people are able to recall to a long sequence of identification names called IP
addresses that computers can interprete. The browsers are only able to connect to the websites that
you'd like to visit once they receive these long number sequences from the name servers.

The browser uses the received number to establish a connection with the web server of the site you
wish to visit. As there are different communication protocols on the web, your computer and the
server has to agree on the "language" that will be used.

Once the connection is established, the client sends a request to the server. This message contains
the address you are trying to reach and specifies what kind of data is needed to display the contents
of the webpage. The period when a webpage is requested is known as navigation start.

RESPONSE

The response of the web page will differ depending on whether the server program was able to find
the requested web page or webfile. Assuming the server was able to locate the web page, it sends the
HTML file to the web browser. The server sends the data which is needed to display the contents of the
web page back to the browser. This ends the communication between the browser and the web server.
The client confirms this by sending a FIN package back to the server. A FIN package is sent from
server or client to terminate a connection after successful transfer of data/establishment of TCP 3-way handshake.

Once the browser has recieved the document(HTML file), it reads it. When a computer reads a file
looking for something, it is called "parsing". The web browser looks at the entire HTML document and
looks for any CSS, JAVASCRIPT and images that are referenced by the page. If resources are found in the
HTML the web browser then requests each of those resource files from the webserver and will be downloaded
by the browser.

BUILD

Once the web browser has the required resources, it can start building the page. The web browser builds
the page by combining the information found in the document (the original HTML file) and the information
found in the resources.
There are basically three steps that the browser takes to build a page:

  • Build the DOM
  • Build the CSSOM
  • Build the render tree

Building the DOM

DOM stands for Document Object Map. It is basically a map of where things are displayed on a page
according to HTML. The DOM represents what the HTML is saying by mapping out the page in a relational manner

Building the CSSOM

CSSOM stands for CSS object map. It is basically a map of what styles should be applied to different
parts of the page according to the CSS. The CSSOM maps out the way things should be presented
using styles.

Building the Render Tree

The render tree essentially takes the DOM and CSSDOM and combines them to create a full map of how
the page will be laid out and presented.

THE RENDER

  • Layout
    This is where the browser determines how big a screen is and how that will affect
    the way the page is displayed.
  • Paint
    Where the browser converts each node in the render tree to actual
    pixels on the screen.
